# CONTEXT PACK

## Project goal
Turn this portfolio into a proof-driven product portfolio for a junior frontend developer.

## Audience
Primary: recruiters and hiring managers
Secondary: founders/clients needing mobile-first frontend work

## Primary CTA
Contact on WhatsApp

## Core positioning
Product-minded junior frontend developer building calm, mobile-first interfaces and clear digital experiences.

## V1 routes
/ home
/work
/work/life-manual
/work/cest-cherife-todo
/about
/contact
/lab

## Visual direction
Simple, clean, calm, product-focused.
Evolve the current warm neutral aesthetic toward a more polished Apple-HIG-inspired feel.
Do not make it loud, flashy, overly dark, or template-like.

## Must keep
- calm premium tone
- strong spacing
- restrained palette direction
- mobile-first behavior
- project-first storytelling

## Must change
- hero must become project-led, not portrait-led
- generic copy must be replaced with proof-driven copy
- broken contact form must be removed or replaced
- projects must communicate value, role, stack, and status
- About section must become a stronger professional summary

## Design rules
- use consistent spacing scale
- use clean typography hierarchy
- keep visual noise low
- prefer real sections over decorative emptiness
- use your photo only in a secondary/supporting role, not dominant hero role

## Accessibility policy
- semantic HTML first
- keyboard-friendly nav and buttons
- visible focus states
- descriptive link/button text
- no color-only meaning

## Maintainability policy
- make minimal diffs
- do not rebuild whole pages when a patch solves it
- do not change unrelated files
- reuse existing structure where possible

## Non-goals
- no framework migration in this phase
- no fake testimonials
- no fake metrics
- no huge animation systems
